How it works

Five plain ideas, doing one job.No buzzwords. Each piece earns its place by saving you a request, a millisecond, or a dollar.

🤖

AI Agent

Reads the question, looks up the answer in your documents, and writes a reply — step by step, like a researcher with a notepad.

🧠

Smart Memory

Finds similar past questions, not just exact matches. "How do refunds work?" and "What's your refund policy?" land on the same answer.

🌐

Shared Memory Across Servers

An optional fast memory store (we use Redis) lets every copy of your app share what it has learned. Nothing is forgotten when a server restarts.

⚡

Live Answer Streaming

You see the answer arrive piece by piece — like watching someone type. No 5-second blank screen.

🐍

One-Line Python SDK

pip install konjoai and you're talking to a real RAG pipeline in five lines. No 200-package framework swamp.

🔐

One Server, Many Customers

Each customer's data — and their cached answers — stay separate. You ship one deployment, not one per tenant.

code

Five lines, working AI agent.You don't need a 200-package framework to get production RAG.

install 
# That's it. Pure SDK. No GPU required client-side.
pip install konjoai
ask a question 
from konjoai.sdk import KonjoClient

with KonjoClient("https://kyro.acme.io", api_key="sk-...") as client:
    response = client.agent_query("What's the refund policy?")
    print(response.answer)
watch the answer arrive live (one step at a time) 
for ev in client.agent_query_stream("How do refunds work?"):
    if ev.type == "step":
        print("  → " + ev.data["action"])
    elif ev.type == "result":
        print("\nAnswer:", ev.data["answer"])
turn on shared memory across servers (one-time setup) 
# Falls back to local memory if your fast store isn't reachable.
export CACHE_ENABLED=true
export CACHE_BACKEND=redis
export CACHE_REDIS_URL=redis://kyro-cache.svc.cluster.local:6379/0
